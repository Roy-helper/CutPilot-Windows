# CutPilot v4.1 手动测试清单

## 1. 中文路径视频导入 + 处理 + 导出

**前提条件：**
- 准备一个正常的 MP4 视频文件（30秒以上，有人声）
- 将视频放入中文名文件夹，例如 `D:\测试素材\电商视频\demo.mp4`
- 输出目录也设置为中文路径，例如 `D:\输出结果\`
- 已在设置页配置好 AI 供应商和 API Key
- ASR 模型已下载就绪

**操作步骤：**
1. 打开 CutPilot，进入设置页，将输出目录改为 `D:\输出结果\`，点击保存
2. 回到工作台，点击"拖入或点击选择视频"，选择 `D:\测试素材\电商视频\demo.mp4`
3. 确认文件列表中显示文件名和 pending 状态
4. 点击"一键生成"，等待处理完成
5. 处理完成后，检查版本卡片是否正常显示（缩略图、文案、标签）
6. 勾选一个版本，点击"导出选中"
7. 打开 `D:\输出结果\demo\` 目录，确认输出文件存在

**预期结果：**
- [ ] 中文路径文件能正常导入，文件名正确显示
- [ ] 处理全程无报错，进度条正常推进
- [ ] 版本卡片显示完整（封面标题 + 文案 + 标签）
- [ ] 输出文件在中文路径目录下正确生成，可正常播放
- [ ] 输出视频音画同步，无花屏或黑屏

---

## 2. 批量处理（含损坏文件）

**前提条件：**
- 准备 2 个正常的 MP4 视频（30秒以上，有人声）
- 准备 1 个损坏文件：创建一个文本文件，将后缀名改为 `.mp4`
- ASR 模型已就绪，AI API 可用

**操作步骤：**
1. 打开工作台，点击导入，同时选中 3 个文件（2个正常 + 1个损坏）
2. 确认文件列表显示 3 个文件，全部为 pending 状态
3. 点击"一键生成"
4. 观察处理过程中每个文件的状态变化

**预期结果：**
- [ ] 损坏文件处理失败，状态变为红色 error，显示具体错误原因
- [ ] 2 个正常文件不受影响，继续处理并成功完成
- [ ] 处理结束后弹出汇总通知："2 成功, 1 失败"
- [ ] 鼠标悬停在失败文件的错误信息上，tooltip 显示完整错误内容
- [ ] 失败文件有红色左边框标识，与成功文件视觉区分明显

---

## 3. 处理中途取消

**前提条件：**
- 准备一个较长的视频（1分钟以上），确保处理时间足够手动操作
- 记下 `%USERPROFILE%\.cutpilot\cache\` 目录当前内容

**操作步骤：**
1. 导入视频，点击"一键生成"
2. 等待进度条推进到 30%~70%（正在剪辑阶段最佳）
3. 点击"取消处理"按钮
4. 打开任务管理器，搜索 `ffmpeg.exe` 进程
5. 检查输出目录是否有不完整的 `.mp4` 文件
6. 检查 `%USERPROFILE%\.cutpilot\cache\` 是否有残留的临时文件

**预期结果：**
- [ ] 点击取消后，进度条消失，状态重置为初始态
- [ ] 文件列表中正在处理的文件状态变为 pending，显示"已取消"
- [ ] 任务管理器中无残留的 `ffmpeg.exe` 进程
- [ ] 输出目录无不完整的 `.mp4` 文件（半成品应被清理）
- [ ] 工作台可以立即重新点击"一键生成"开始新的处理

---

## 4. 拖放导入视频

**前提条件：**
- 工作台处于空状态（无已导入文件）
- 准备 2-3 个视频文件在文件管理器中

**操作步骤：**
1. 打开 CutPilot 工作台，确认显示空状态拖放区域
2. 从文件管理器拖动 1 个 MP4 文件到拖放区域上方（不松手）
3. 观察拖放区域的视觉变化
4. 松开鼠标
5. 再次拖入 2 个文件（批量拖放）
6. 尝试拖入一个 `.txt` 文件

**预期结果：**
- [ ] 空状态显示"拖入或点击选择视频"文案
- [ ] 拖入悬停时，区域变为蓝色高亮，文案变为"松开即可导入"
- [ ] 松开后文件成功添加到文件列表，状态为 pending
- [ ] 批量拖放多个文件均成功导入
- [ ] 非视频格式文件（.txt）被忽略，不添加到列表
- [ ] 拖入后拖放区域被文件列表替代

## 5. 设置页修改画质后重新处理

**前提条件：**
- 已有一个可正常处理的视频
- 当前画质设置为默认（4K）

**操作步骤：**
1. 进入设置页，将画质从 4K 改为 720P，点击保存
2. 回到工作台，导入视频，点击"一键生成"
3. 处理完成后，右键输出文件 → 属性 → 详细信息，查看分辨率
4. 回到设置页，将画质改回 4K，保存
5. 清空工作台，重新导入同一视频，再次处理
6. 对比两次输出文件的分辨率和文件大小

**预期结果：**
- [ ] 设置修改后点击保存，提示保存成功
- [ ] 离开设置页时若有未保存修改，弹出确认提示
- [ ] 720P 输出文件分辨率不超过 1280x720，文件体积明显小于 4K
- [ ] 4K 输出文件分辨率为源视频原始分辨率（最高 3840x2160）
- [ ] 两次处理均正常完成，无报错

## 6. GPU 编码器状态显示

**前提条件：**
- 系统已安装 NVIDIA/AMD/Intel 显卡驱动（或纯 CPU 环境）
- FFmpeg 已安装且在 PATH 中

**操作步骤：**
1. 打开 CutPilot，进入设置页
2. 找到"导出引擎"区域，查看显示内容
3. 在工作台底部 footer 查看编码器信息
4. （可选）在命令行运行 `ffmpeg -encoders | findstr nvenc` 对比确认

**预期结果：**

*有 NVIDIA GPU 的环境：*
- [ ] 设置页显示绿色圆点 + "NVENC (NVIDIA GeForce RTX XXXX)" + 并行 N 路
- [ ] 工作台底部显示 "编码器: NVIDIA NVENC"

*纯 CPU 环境：*
- [ ] 设置页显示黄色圆点 + "CPU (libx264)"
- [ ] 工作台底部显示 "编码器: Software x264"

*通用：*
- [ ] 显示的编码器与 `ffmpeg -encoders` 的实际可用编码器一致
- [ ] 并行路数显示合理（GeForce 消费卡为 3，专业卡为 8）

---

## 7. 版本卡片文案复制

**前提条件：**
- 已成功处理一个视频，工作台显示版本卡片（至少 2 个版本）

**操作步骤：**
1. 在工作台点击顶栏"更多"按钮，选择"复制全部文案"
2. 打开记事本，粘贴内容
3. 检查粘贴内容的格式和完整性
4. 在单个版本卡片上点击复制按钮（如果有）
5. 粘贴并检查

**预期结果：**
- [ ] "复制全部文案"在"更多"下拉菜单中可见且可点击
- [ ] 粘贴内容包含所有版本，格式为：
  ```
  === V1 [切入角度] ===
  发布文案: ...
  封面主标题: ...
  封面副标题: ...
  标签: #tag1 #tag2 ...
  ```
- [ ] 每个版本的文案、标题、标签均完整，无乱码
- [ ] 复制成功后有 toast 通知提示
